一、P0 总目标
P0 只做一件事：

打通“txt 小说 → 章节 → scene → VN 脚本 → 预览播放器”的最小可运行闭环。

所以 P0 的任务必须围绕这个闭环服务。
凡是不能直接帮助闭环成立的，都不应该进入 P0。

二、P0 模块总览
建议 P0 拆成 8 个模块：

项目与输入模块
结构解析模块
文本主链路 Agent 模块
状态流转与调度模块
缓存与重跑模块
预览播放器模块
工作台基础页面模块
配置与运行环境模块
三、模块 1：项目与输入模块
目标
支持创建项目并导入 txt 小说，形成后续处理的最小项目对象。

任务清单
1.1 项目数据骨架
定义：

Project 基础信息
文件信息
当前状态
参数配置
创建时间/更新时间
1.2 txt 文件导入
支持：

选择本地 txt 文件
读取文件
保存原始文本或文件引用
记录编码信息
1.3 项目创建流程
支持：

输入项目名
默认参数初始化
创建 ProjectState
1.4 项目列表最小版
支持：

展示项目列表
显示项目状态
进入项目
1.5 项目总览最小版
支持：

展示项目基础信息
展示总进度
展示章节处理概览
依赖关系
几乎无上游依赖
是所有后续模块入口
验收标准
能创建项目
能导入 txt
能在项目列表中看到项目
能进入项目总览页

目标

任务清单

包括：

编码统一
空白规范化
基础噪声清洗

2.2 章节标题识别
支持识别常见格式：

第一章
数字章节
番外
后记
2.3 章节正文切分
将文本按章节边界切开
生成章节列表
2.4 特殊章节标记
番外
后记
作者的话（如果能识别）
2.5 Structure Agent 最小版封装
输入原始文本

十二、模块 5：缓存与重跑模块
目标
避免重复调用模型，并支持局部重跑。

任务清单
5.1 结果缓存设计
缓存至少包括：

structure result
parsing result
attribution result
segmentation result
scene script
fidelity report
5.2 缓存读取逻辑
若命中缓存则跳过重复执行
显示缓存命中状态（内部即可）
5.3 手动重跑单章
重新执行某章全流程
5.4 手动重跑某阶段
例如：

从 parsing 重跑
从 segmentation 重跑
从 mapping 重跑
5.5 手动重跑单个 scene
重跑 mapping
重跑 review
5.6 缓存失效规则最小版
改 segmentation strategy → segmentation 以下失效
改 mapping strategy → mapping 以下失效
改 fidelity mode → review 失效
验收标准
同一章重复运行不会重复浪费调用
用户可重跑失败章节
scene 级重跑可用

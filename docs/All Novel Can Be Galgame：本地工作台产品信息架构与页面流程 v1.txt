一、产品形态
建议把本地工作台定义成：

项目制、左侧导航、中心工作区、右侧详情/日志面板 的桌面式工作台。

它更像：

创作工具
IDE
游戏制作工具
而不是一个简单的“上传文件 → 等结果”的网页。

二、产品信息架构总览
第一层：全局层
解决“项目管理”和“系统配置”。

包括：

项目列表页
新建项目页 / 导入流程
设置页
模型/API 配置页
第二层：项目工作台层
解决“单个小说项目”的处理、查看、预览。

包括：

项目总览
原文结构页
章节处理页
Scene 工作区
VN 脚本页

预览播放器页
任务与日志页

五、页面 1：项目列表页
页面目标
让用户看到自己有哪些项目，并快速进入或新建。

页面内容
每个项目卡片建议显示：

项目名
原始小说文件名
创建时间
最近更新时间
当前状态
章节数
已完成章节数
预览可用状态
页面操作
新建项目
打开项目
删除项目
复制项目
查看最近日志
设计重点
项目列表页不要太复杂，它只是入口。

六、页面 2：新建项目 / 导入流程
页面目标
完成：

导入小说
设置基础参数
创建项目
页面流程建议
采用 3 步式向导

Step 1：导入文件
选择 txt 文件
显示文件名
显示大小
显示编码检测结果（可选）
Step 2：项目基础信息
项目名称（默认取文件名）
小说标题（可编辑）
简介（可选）
标签（可选）
Step 3：处理参数
忠实度模式：保守 / 标准
scene 切分策略：保守 / 标准
视觉风格模板：校园恋爱 / 都市恋爱 / 清新日系
预算模式：高质量 / 平衡 / 省钱
设计重点
不要一开始暴露太多高级参数。

页面目标

十九、交互原则
默认简单，细节可展开
不要对普通用户暴露“Agent”术语
失败信息必须可理解
视觉层不阻塞主流程
所有复杂流程都回到“章节”

三、阶段 1：基础清洗
目标
让所有 txt 至少达到“可稳定解析”的程度。

常见问题
编码不统一（UTF-8 / GBK / ANSI）
空行混乱
全角半角混杂
引号风格不统一
重复标题
广告/网站水印
作者注、PS、碎碎念
平台残留
番外/正文混杂
章节标题格式不统一
清洗目标
统一成：

UTF-8 编码
换行规范
空白规范
引号规范
去除明显平台噪声
保留原始正文内容顺序
不改写正文文字
清洗原则
可以做的
去广告
去平台尾巴
去明显重复空行
统一标点形式
统一换行
不可以做的
自动润色
自动改错别字
自动补句子
改章节顺序
删作者正文中的有效内容

六、阶段 4：内容分析与分层
目标
知道“它适合拿来打磨哪个 Agent”。

需要分析的内容
对话密度
心理描写密度
动作描写密度
场景描写密度
角色密度
场景切换频率
为什么要做这些分析
因为不同 Agent 需要不同类型的样本。

例如：

Narrative Parsing Agent 需要对话、旁白、心理、动作混合丰富的文本
Attribution Agent 需要连续对话多、说话人省略多、代词复杂的文本
Scene Segmentation Agent 需要章节内部有明显场景变化的文本
Visual Prompt Extractor 需要外貌、服装、环境描写较丰富的文本

八、抽样建集方案
1. 开发集（Development Set）
20~40 本书
每本抽若干章节
用于 prompt 调整、规则调试、错误分析
2. 验证集（Validation Set）
10~20 本书
不与开发集重叠
用于比较版本优劣、防止过拟合
3. 压力测试集（Stress Test Set）
10 本左右
覆盖：
连续对话不写说话人
大量心理描写
章节超长
场景切换频繁
代词模糊
别名复杂
回忆插叙多
4. 人工标注金标准集（Gold Set）
30~60 个章节
或 100~200 个 scene 级片段
人工精标
覆盖核心任务
